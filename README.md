# iOS（SwiftUI）工程说明

本目录提供 **iOS 15+** 的纯离线单模型 Agent 工程骨架，用于逐步接入 Gemma 3n E2B 的真实推理链路、SQLCipher+A ES-256 加密存储，以及 FAISS Lite 向量记忆。

## 生成并打开工程

本工程使用 **XcodeGen** 生成 `.xcodeproj`（避免手写 pbxproj）。

1) 安装 XcodeGen（任选其一）：

- `brew install xcodegen`
- 或使用你们内部的构建环境安装方式

2) 在仓库根目录执行：

```bash
cd ios
xcodegen generate
open OfflineAgent.xcodeproj
```

## 当前状态（用于推理Spike）

- `OfflineAgentCore` 提供 `InferenceService`（支持 **streaming/cancel**）与 `MockInferenceBackend`（可运行的流式输出 mock）。
- 后续拿到 iOS 可用的 Gemma 3n E2B 模型产物后，将新增真实 backend（CoreML 或 llama.cpp 等）替换 mock，即可完成推理Spike验收。

纯离线AI Agent应用开发文档
编制时间：2026年3月16日
核心开发指令：唯一模型Gemma 3n E2B，INT4/INT8分级量化，保留向量库+SQLite+AES-256核心模块，记忆数据加密永久保存10年以上，纯离线开发适配Google Play/App Store双平台合规，所有工具能力由单模型统一承载，无任何冗余模型及切换逻辑。
核心原则：保核心、砍鸡肋、松指标、增实用、省工期，核心功能全留，砍掉难开发/非刚需细节，放宽无感知数字指标，新增实用功能并预留升级接口，确保40天核心开发落地，适配双平台上架要求，全程纯离线、低功耗、易开发。
一、项目概述
1. 项目目标
开发一款纯离线本地运行的AI Agent (Offline) 应用，实现工作+情感双模式双助理核心能力，具备前置主动交互、永久记忆串联等特性；首轮砍掉本地文件智能整理功能（后续版本迭代添加），满足全球隐私合规标准，适配Google Play / App Store平台上架要求。
核心约束：
•	全功能基于单模型Google Gemma 3n E2B承接，剔除百度PP-OCRv5-Slim及其他额外轻量工具模型适配开发；
•	模型采用INT4/INT8量化分级方案，INT4为低配机兜底、INT8为标准落地版本；
•	功耗/发热不硬卡数字，以用户摸着手不烫、耗电不明显为准；
•	记忆数据双引擎加密存储，永久保存10年以上，无自动清理逻辑。
2. 开发阶段
遵循标准化研发流程，分阶段落地：需求确认→技术选型与架构设计→模块开发实现（40天）→单元/集成测试→合规审核→部署上架。
3. 核心资源配置
核心开发工程师需完成以下工作：
•	统筹单模型Gemma 3n E2B手机端适配、INT4/INT8量化压缩与推理逻辑优化；
•	完成Android/iOS原生开发、本地数据库搭建、功能模块开发；
•	实现极简界面还原与交互、UI/UX落地衔接；
•	配合隐私合规适配，支撑全维度测试；
•	实现记忆数据10年以上永久存储逻辑，剔除多模型适配、切换相关开发。
二、核心功能需求
1. 双模式双助理
工作模式
依托单模型实现高效任务处理、2K修图（仅划痕修复，砍掉超清放大/模糊修复）、多语言离线翻译，输出专业严谨；2K修图由单模型统一承载，内存按INT4/INT8分级管控。
情感模式
贴心陪伴、情绪安抚，对话拟人自然，配合语音包实现沉浸式交互，支持语气柔和度适配；语音交互由单模型统一承载，无独立ASR/TTS模型。
模式切换
仅支持关键词触发（“翻译/修图”切工作，“聊天/心情不好”切情感），砍掉上下文智能猜场景功能；切换无卡顿、界面即时同步，仅单模型内逻辑切换。
2. 前置思维主动交互
AI自主发起对话，无需用户先提问，交互频率可配置，核心要求：
•	主动提醒：待办任务、重要节点、使用习惯相关定时/触发式提醒，核心提醒必响必弹；
•	主动延续话题：仅记“时间+事件”关键词接话，砍掉久远记忆无头绪接话，话题记忆永久留存；
•	频率配置：支持「高/中/低/关闭」四档，默认中低档位。
3. 长期永久记忆系统
基于轻量向量库+SQLite双引擎本地存储，全程AES-256加密，永久保存10年以上，无自动清理/去重/过滤逻辑，核心要求：
•	砍掉智能记忆清洗、优先级打标功能，确保数据完整；
•	跨会话关联记忆上下文，记忆召回响应时间≤2秒，能力由单模型统一承载；
•	支持用户手动一键清空所有记忆，清空后数据不可恢复。
4. 纯离线本地运行
全程无云端交互，数据不上云，符合双平台合规，核心要求：
•	无网络依赖：无网状态下正常启动、使用所有功能，单模型全程本地运行；
•	本地存储：聊天记录、记忆数据、模型文件等仅保存在设备本地，AES-256加密，符合GDPR/CCPA/CPRA隐私法规。
5. 多语言翻译
基于单模型纯离线实现，无额外翻译模型，核心要求：
•	支持中英/中日/中韩/中西等8种主流语言互译；
•	翻译精准流畅，支持整段/短句输入，响应≤3s，翻译历史同步至永久记忆。
6. 轻量工具集（单模型统一承载）
剔除所有独立轻量工具模型，原能力由单模型按需调用，全程驻留不发热，包含：
•	2K修图（仅划痕修复）：调起相册无延迟，内存分级管控，操作记录本地留存；
•	语音包交互：单模型承载本地ASR+TTS，支持男女双音色、语速调节，识别准确率≥90%，记录同步至记忆；
•	文档文字增强：输入指令触发，支持扫描件/截图去阴影、折痕、文字锐化，结果仅本地保存。
7. 手机端技术保障
•	硬件加速：深度适配手机NPU/GPU，调用硬件加速接口，优先CPU推理控制发热，仅做单模型加速优化；
•	流畅运行：对话/工具使用无延迟，中低端机型无明显掉帧，量化版本自动适配硬件；
•	内存管控：分场景限制内存，无泄漏，单模型INT4/INT8分级管控，永久存储模块不占用额外运行内存。

五、技术架构设计
1. 整体架构分层（从下至上，纯本地单模型，模块化低耦合）
剔除所有多模型相关分层，新增量化版本管理服务，重点实现永久记忆存储，贴合双平台落地规范，共5层：
（1）硬件适配层
封装NPU/GPU/CPU硬件加速接口（Qualcomm SNPE/NNAPI/Core ML），为单模型提供统一支撑；按硬件配置适配INT4/INT8版本，优先CPU推理控制发热，剔除多模型加速适配。
（2）模型层
仅包含单模型Gemma 3n E2B（INT4/INT8分级量化），为架构核心，统一承载所有能力（对话、翻译、修图、ASR/TTS、文字增强等）；实现量化版本按需加载+后台轻量驻留，无模型切换，中文场景降级处理。
（3）核心服务层
封装核心能力，提供统一接口，模块低耦合，剔除多模型调度相关模块，新增量化版本管理服务、开关服务，核心模块：
•	对话服务：双模式逻辑、主动交互、上下文串联、逐字输出速度调节，基于单模型流式推理；
•	记忆服务：双引擎存储、记忆检索/召回，与单模型直连，实现10年以上加密永久存储；
•	工具服务：单模型工具能力分发，精准指令触发逻辑处理；
•	合规服务：权限管理、AES-256加解密、日志管理、隐私保护；
•	开关服务：喇叭/小铃铛状态切换、记忆、声音/弹窗控制；
•	量化版本管理服务：硬件检测、INT4/INT8自动/手动切换、版本更新、运行状态监控，CPU优先推理。
（4）UI交互层
实现极简全屏沉浸式对话界面，封装顶栏/对话区/输入栏组件；处理用户所有交互操作，衔接核心服务层；仅做单模型内模式切换，剔除多模型交互逻辑。
（5）应用层
整合各层能力，实现APP整体功能（启动、版本更新、异常处理）；新增量化版本兜底、存储模块异常防护逻辑；为后续功能预留接口，剔除多模型管理逻辑，符合双平台运行规范。
