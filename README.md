# AI Agent (Offline) 开发文档
---

## 一、项目概述

### 1. 项目目标

开发一款纯离线本地运行的 AI Agent (Offline) 应用，实现工作 + 情感双模式双助理核心能力，具备前置主动交互、永久记忆串联等特性，首轮砍掉本地文件智能整理功能（后续版本迭代添加），满足全球隐私合规标准，可适配 Google Play / App Store 平台上架要求。

**核心约束**：
- 全功能基于单模型 **Google Gemma 3n E2B** 承接实现，剔除百度 PP-OCRv5-Slim 及其他额外轻量工具模型适配开发，原工具能力由单模型统一承载；
- 模型采用 **INT4/INT8 分级量化压缩**方案，INT4 量化为低配机兜底、INT8 量化为标准落地版本，保障手机端运行流畅性；
- 不硬卡 “低功耗、不发热” 死数字，以用户摸着手不烫、耗电不明显为准。

### 2. 开发阶段

项目遵循标准化研发流程，分阶段落地：需求确认 → 技术选型与架构设计 → 模块开发实现（40 天） → 单元 / 集成测试 → 合规审核 → 部署上架。

### 3. 核心资源配置

- **核心开发工程师**：统筹负责单模型 Gemma 3n E2B 手机端适配、INT4/INT8 量化压缩与推理逻辑优化；完成 Android/iOS 原生开发、本地数据库搭建、功能模块开发；负责极简界面还原与交互实现、UI/UX 落地衔接；配合隐私合规适配技术对接，协助开展兼容性、性能、稳定性、隐私安全全维度测试技术支撑；剔除多模型适配、模型间切换相关开发工作。

## 二、核心功能需求

### 1. 双模式双助理

- **工作模式**：依托单模型 Gemma 3n E2B 实现高效任务处理、2K 修图（仅做划痕修复，砍掉超清放大 / 模糊修复）、多语言离线翻译，输出专业、严谨、高效，贴合职场刚需；2K 修图能力由单模型统一承载，无独立修图模型，单模型量化后运行内存按 INT4/INT8 分级管控，更省电不发热。
- **情感模式**：贴心陪伴、情绪安抚，对话语气拟人自然，配合语音包实现沉浸式交互，支持语气柔和度适配；语音交互能力由单模型统一承载，无独立 ASR/TTS 工具模型。
- **模式切换**：仅支持关键词触发切换（说 “翻译 / 修图” 切工作，说 “聊天 / 心情不好” 切情感），砍掉上下文智能猜场景功能；切换无卡顿、界面即时同步；剔除多模型切换相关逻辑，仅做单模型内模式逻辑切换。

### 2. 前置思维主动交互

AI 自主发起对话，无需用户先提问，实现主动式助理体验，交互频率可配置：
- **主动提醒**：待办任务、重要节点、用户使用习惯相关定时 / 触发式提醒，任务 / 文档整理完成后自动弹窗通知，闹钟 / 事务提醒必响、必弹，核心功能不删减。
- **主动延续话题**：仅记 “时间 + 事件” 关键词接话（如记 “明天谈客户” 次日主动询问），砍掉久远记忆无头绪接话题功能，无交互断层，不强行打扰。
- **可配置交互频率**：支持用户设置「高 / 中 / 低 / 关闭」四档主动交互频率，默认中低档位。

### 3. 长期永久记忆系统

轻量向量库 + SQLite 双引擎本地存储，数据加密稳定保存 10 年以上，无冗余不卡顿：
- 砍掉智能记忆清洗、记忆优先级打标功能，不自动去重 / 过滤信息，不区分记忆重要程度。
- **历史串联**：跨会话关联记忆上下文，实现真人式连贯对话，记忆召回响应时间放宽至 ≤2 秒；记忆检索 / 处理能力由单模型 Gemma 3n E2B 统一承载。
- **本地加密**：所有记忆数据 AES-256 加密存储，支持用户一键清空所有记忆，清空后不可恢复。
- 保留双引擎加密存储、记忆存 10 年、跨会话记内容的核心功能。

### 4. 纯离线本地运行

全程无任何云端交互，数据不上云，隐私绝对安全：
- **无网络依赖**：无网状态下可正常启动、对话、使用所有功能，无功能限制；单模型全程本地运行，无模型云端调用、无跨模型数据交互。
- **本地存储**：聊天记录、记忆数据、文件索引、单模型文件均仅保存在用户设备本地，无任何隐性上传。

### 6. 多语言翻译

完全基于单模型 Gemma 3n E2B 实现，无需额外翻译模型，纯离线可用：
- 支持主流语言离线互译（中英 / 中日 / 中韩 / 中西等 8 种常用语言）。
- 翻译精准、语句流畅，支持整段 / 短句输入，响应 ≤3s，满足办公、出行刚需。

### 7. 轻量工具集（单模型统一承载，无独立工具模型）

剔除所有独立轻量工具模型开发，原工具能力由单模型 Gemma 3n E2B 按需调用，无模型加载 / 释放切换逻辑，单模型全程驻留（量化后低内存占用），不发热、低内存消耗：
- **2K 修图（仅划痕修复）**：由单模型 Gemma 3n E2B 直接承载实现，调起相册无延迟，模型内存按 INT4/INT8 分级管控，单模型做统一加密校验。
- **语音包交互**：由单模型 Gemma 3n E2B 直接承载本地 ASR 语音识别 + TTS 语音合成能力，支撑情感模式 / 语音对话，支持男女双音色、语速调节，语音识别准确率放宽至 ≥90%。
- **文档文字增强**：输入「文字增强」触发，支持扫描件 / 截图去阴影、去折痕、文字锐化，由单模型 Gemma 3n E2B 统一实现。

### 8. 手机端技术保障

- **本地 AI 硬件加速**：深度适配手机 NPU/GPU，调用硬件加速接口实现单模型 Gemma 3n E2B 推理加速，减少 CPU 占用；剔除多模型硬件加速适配逻辑，仅做单模型硬件加速优化。
- **全程流畅不卡顿**：对话 / 工具使用无延迟，无模型切换卡顿，中低端机型无明显掉帧。
- **内存占用严格管控**：分场景限制内存，长期运行无内存泄漏，保障手机其他应用正常使用；单模型按 INT4/INT8 分级量化，分别管控内存占用，无多模型内存叠加消耗。

## 三、极简界面交互设计（UI/UX 核心规范）

### 1. 设计理念

极致极简、无冗余元素、沉浸式对话、单手可操作，核心功能零学习成本：
- 无广告、无推送、无复杂菜单 / 二级页面。
- 所有功能嵌入聊天界面，仅支持精准指令触发，砍掉模糊识别功能（必须说 “修图 / 翻译 / 设置” 等明确指令）；指令触发仅调用单模型对应能力，无模型切换逻辑。
- 工作模式简洁克制，情感模式柔和温暖，模式切换视觉即时同步。

### 2. 界面布局（全屏沉浸式）

```
[顶部状态栏：仅显示模式 + 隐私状态 + 双开关]
极简顶栏 (高度 40dp)
工作 / 情感模式 | 本地离线安全 | 喇叭开关 | 小铃铛开关

[对话区域]
← 全屏沉浸式 (占比 90%)

[输入框 (仅一行)]
极简输入栏 (固定底部)
```

### 3. 核心 UI 组件规范

#### （1）顶部导航栏（高度 40dp，极简化，新增双开关入口）
- **左侧**：模式标签，点击可一键切换模式
  - 工作模式：白底黑字 + 「工作」简洁文字标签，无多余样式
  - 情感模式：浅柔渐变底（浅粉 / 浅蓝可选） + 「陪伴」文字标签，视觉柔和
- **中间**：隐私状态固定展示
  - 锁形图标 +「本地离线」文字，全程展示，强化用户隐私安全感；新增「单模型本地运行」小字标识，贴合技术落地，强化用户感知
- **右侧**：新增喇叭总开关、小铃铛开关并列入口，图标式展示，点击一键切换，状态记忆（下次打开 APP 保持上次设置）。
- 无菜单、无头像、无其他冗余按钮，设置功能通过输入指令触发。

#### （2）对话气泡区域（全屏占比 90%，沉浸式呼吸感交互）
- **工作模式气泡**：纯白底 + 浅灰细边框，文字左对齐、行间距 1.2，简洁干练，无圆角 / 轻微圆角
- **情感模式气泡**：浅柔渐变底，文字居中 / 左对齐，行间距 1.4，圆角 12dp，视觉温暖
- 无头像、无名称、无时间戳，极致极简，减少视觉干扰
- **AI 消息输出**：实现呼吸感逐字输出，速度按手机好坏自动调（中低端机快一点，避免卡顿），模拟真人说话节奏，非一次性弹出；砍掉小铃铛轻量提示图标，整合至顶部小铃铛开关功能；逐字输出由单模型流式推理实现，无多模型数据拼接逻辑。

#### （3）输入栏（固定底部，一行高度，单手可操作）
- **左侧**：语音按钮（麦克风图标），点击触发语音输入，长按可快捷发送语音；语音输入由单模型直接识别，无独立 ASR 模型调用
- **中间**：单行输入框，无多余样式，默认提示词「输入精准指令 / 对话」，极简占位符
- **右侧**：发送按钮（火箭图标），点击发送文字，无其他功能
- 输入框支持快捷唤醒功能：输入「/」弹出常用指令快捷栏（翻译 / 修图 / 设置）；所有快捷指令均触发单模型对应能力，无模型切换。

#### （4）功能入口（全部藏于输入框精准指令，无单独模块）
- 无需单独页面、无需 Tab 栏、无悬浮窗，所有功能通过精准自然语言指令触发。
- **核心指令触发逻辑（仅精准匹配）**：
  - 输入「翻译」→ 直接开启翻译功能弹窗，输入待译文字即可；由单模型 Gemma 3n E2B 实现翻译
  - 输入「修图」→ 直接调起手机相册，选择照片即可进行 2K 划痕修复；由单模型 Gemma 3n E2B 实现修图
  - 输入「设置 / 主动交互 / 记忆清空」→ 调起对应极简设置弹窗
- 所有功能操作结果以卡片消息形式在对话区展示，操作完成后返回对话状态，用完即走。
- 文档文字增强功能：输入「文字增强」触发，支持扫描件 / 截图去阴影、去折痕、文字锐化，由单模型 Gemma 3n E2B 实现。

### 4. 交互逻辑（极简零学习成本，全程单手可操作）

- 打开 APP → 直接进入对话页（无引导页、无首页、无加载广告，启动即对话）；APP 启动仅加载单模型 Gemma 3n E2B（量化版本），无多模型加载等待
- AI 主动触发交互（无需用户点击 / 输入，首次启动主动问候，后续按配置频率触发）
- 单手操作全覆盖：所有按钮 / 输入框 / 双开关在手机单手操作区域，支持点击、长按、滑动简单操作
- 模式一键切换：点击顶部模式标签即时切换，界面 / 气泡样式同步变更，无延迟；仅为单模型内的模式逻辑切换，无模型加载 / 切换耗时
- 无二级页面：所有功能 / 设置在当前对话页以弹窗 / 卡片完成，不跳转新页面，操作闭环
- **双开关核心交互**：
  - 喇叭总开关：控制 APP 所有声音（AI 播报、提示音、事务提醒、对话文字语音），开 = 全响，关 = 全静，仅控制声音，消息不丢失；声音提示用柔和提示音，不刺耳，与 AI 语音音量同步；AI 语音由单模型 TTS 能力实现。
  - 小铃铛开关：控制消息 / 事务提醒（聊天消息、闹钟、待办）的系统弹窗与震动，开 = 收通知 / 震动，关 = 屏蔽弹窗 / 震动，消息仍保留在 APP 内，可查看。

### 5. 视觉风格规范

- **配色**：无高饱和色，整体柔和低对比，适配手机护眼模式
  - 工作模式：黑白灰 + 深蓝浅点缀（专业冷静），无渐变 / 特效
  - 情感模式：米白底色 + 浅粉 / 浅蓝柔渐变（温暖治愈），渐变低饱和度
- **字体**：系统默认无衬线字体，字号可配置（默认 16sp），清晰易读，无特殊字体
- **动画**：仅保留轻量必要动效，不花哨、不耗电、不卡顿
  - AI 消息呼吸感逐字输出，无其他弹窗动画
  - 模式切换 / 功能弹窗淡入淡出，过渡时长 ≤0.3s
  - 双开关切换仅做轻量图标变色动效，无复杂动画
- **无障碍**：支持系统大字体、高对比度模式，适配屏幕朗读，按钮 / 输入框 / 双开关有焦点提示。

## 四、非功能需求

### 1. 性能指标

- **启动速度**：冷启动 ≤3 秒，热启动 ≤1 秒，无卡顿、无白屏、无启动广告，启动后直接进入对话界面，符合极简交互需求；冷启动仅加载单模型量化版本，无多模型加载耗时
- **模型推理速度**：依托手机 NPU/GPU 硬件加速，单模型 Gemma 3n E2B（INT8 标准版本）推理响应时间 ≤500ms，INT4 兜底版本 ≤800ms，AI 消息呼吸感逐字输出无断档；无轻量工具模型加载时间，单模型全程驻留，运行时无明显卡顿，调起 / 使用无延迟。
- **内存占用**：严格管控各场景内存消耗，确保不影响手机其他应用正常运行；其中单模型 Gemma 3n E2B INT8 量化版本运行时内存占用 ≤400MB，INT4 量化兜底版本 ≤250MB，APP 后台运行时内存占用 ≤80MB；实现单模型量化版本按需加载（低配机自动切 INT4）、后台轻量驻留，无模型释放 / 重新加载逻辑，实时清理内存碎片，无内存泄漏。
- **功耗控制**：连续使用 1 小时，手机耗电量放宽至 ≤20%；待机 / 后台运行时功耗趋近于 0，无后台偷跑 / 唤醒；单模型运行时机身温度放宽至 ≤42℃（仅温热，不烫手），实现低功耗、无明显发热的核心约束；无多模型同时运行的功耗叠加，单模型推理做功耗优化。
- **稳定性**：连续运行 72 小时无崩溃、无闪退、无 ANR；极端场景（低电量、内存不足、无网）下，优先保障核心对话功能正常，工具功能优雅降级（提示而非崩溃），不影响基础使用；新增量化版本自动切换兜底方案，若 INT8 版本卡顿 / 内存不足，APP 自动切至 INT4 量化版本，所有功能不变，仅推理精度轻微下调。
- **兼容性**：适配 Android 10.0 及以上、iOS 15.0 及以上版本手机，覆盖海外主流中低端机型（骁龙 6/7 系列、天玑 700/800 系列）；支持横屏 / 竖屏自动切换，切换后界面布局无错乱、气泡无重叠，功能正常使用；单模型做全机型量化版本适配，无多模型机型兼容差异。

### 2. 隐私合规需求

全程遵循全球隐私合规标准，适配 Google Play、App Store 上架要求，核心聚焦本地隐私保护，无任何云端数据交互：

- **数据存储**：所有用户数据（聊天记录、记忆数据、文件索引、操作日志）、单模型 Gemma 3n E2B 文件仅存储在用户设备本地，采用 AES-256 加密算法加密存储，无任何数据上传云端、无数据共享、无数据统计行为。
- **权限管理**：仅申请 3 项必要权限，无多余权限申请，权限申请时弹窗使用统一话术：开启权限，让 AI 助手在息屏、后台、通话时正常播报声音。所有数据仅在本地处理，不上传；用户拒绝非核心权限不影响基础对话功能：
  - **存储权限**：仅用于修图 / 文字增强时读取 / 保存照片，无其他用途；修图 / 文字增强由单模型处理，无独立模型访问照片
  - **麦克风权限**：仅用于语音交互 / 语音输入，无语音数据本地录制 / 上传；语音输入由单模型本地识别，无数据缓存
  - **相机权限**：仅用于修图时直接拍照，无相机预览 / 录制行为
- 支持用户随时在 APP 设置 / 系统设置关闭权限，关闭后对应功能禁用，核心对话功能正常；确保息屏、后台、打电话时，系统允许 APP 声音输出 / 弹窗提醒，不滥用权限。
- **合规适配**：符合 GDPR（欧盟隐私法规）、CCPA（加州消费者隐私法）、CPRA 等海外主流隐私法规，提供符合平台要求的隐私政策、用户协议，明确数据使用范围 / 存储方式 / 用户权利；支持用户一键清空所有本地数据（聊天记录 + 记忆 + 文件索引 + 模型缓存），清空后数据不可恢复，保障用户数据绝对控制权。
- **日志管理**：保留日志本地导出功能，新增独立功能入口，操作日志仅本地存储，用于开发问题排查，日志内容不包含用户敏感信息（无聊天内容 / 记忆数据），日志文件大小限制 ≤100MB，支持用户手动删除 / 本地导出所有日志；无日志上传、无日志统计、无远程日志查看行为。

### 3. 可维护性需求

- **代码规范**：严格遵循 Android/iOS 原生开发规范，代码注释完整（关键逻辑、核心算法、参数定义、接口调用均添加清晰注释），代码结构模块化、低耦合，便于后续迭代优化、问题排查、功能扩展；剔除多模型相关代码模块，仅保留单模型适配、量化、推理相关代码，简化代码结构。
- **版本迭代**：支持 OTA 在线更新（仅更新 APP 安装包，不涉及任何用户本地数据上传 / 读取），更新过程中不丢失用户本地数据，更新后模型 / 功能正常使用；预留单模型 Gemma 3n E2B 量化版本本地离线更新接口，支持 INT4/INT8 版本独立更新，更新后无需重新安装 APP；为砍掉 / 简化的功能（4K 修图、文件整理、应用锁）预留升级接口，后续添加无需重写代码。
- **问题排查**：内置本地日志记录模块，仅记录崩溃信息、异常行为、功能调用日志，无敏感信息；支持用户手动导出日志文件（本地保存），便于开发工程师排查问题；核心模块预留单模型量化版本调试接口，调试模式仅支持开发者本地开启，不对外暴露、无远程调试功能。

### 4. 安全性需求

- **本地数据安全**：砍掉 APP 应用锁、后台自动锁数据功能（后续版本迭代添加）；仅保留 AES-256 加密存储，依托系统权限保障数据安全，避免额外开发工作量；单模型文件与用户数据统一采用 AES-256 加密，无单独模型加密逻辑。
- **模型安全**：单模型 Gemma 3n E2B（INT4/INT8 量化版本） 本地存储，模型文件采用加密校验机制（MD5 校验），防止模型被篡改、盗用、提取；无独立工具模型加密校验，单模型做全局加密校验，模型加载时自动进行完整性校验，校验失败则禁止启动 APP，并弹窗提示，保障模型运行安全。
- **输入安全**：对用户文字 / 语音输入内容进行过滤，防止恶意输入、代码注入、特殊字符攻击，避免 APP 崩溃或本地数据被破坏；输入过滤不影响正常对话 / 精准指令使用，无过度过滤；语音输入采用本地实时识别，识别后即时释放语音缓存，无语音数据本地保存；输入过滤由单模型前置逻辑实现，无多模型输入校验。

### 5. 易用性需求

- **零学习成本**：界面极简、操作逻辑简单，用户打开 APP 即可直接对话 / 使用功能，无需阅读使用说明、无需进行复杂设置；所有功能通过精准自然语言指令触发，符合用户日常说话习惯；所有指令均指向单模型能力，无模型选择 / 切换操作，降低用户学习成本。
- **容错性**：用户输入错误 / 无效指令时，AI 主动弹窗提示正确精准指令示例（如「你可以输入：翻译 / 修图 / 文字增强」），不返回无意义内容；操作失误（如误触发送、误删消息、误点修图）时，支持一键撤回 / 取消，降低用户操作成本。
- **无障碍支持**：全面适配系统无障碍功能，支持屏幕朗读（所有按钮 / 输入框 / 气泡 / 双开关有朗读提示）、高对比度模式、大字体模式；语音交互支持调节语速（慢 / 中 / 快）、音量，TTS 合成语音清晰可辨，满足不同用户使用需求；语音交互所有能力由单模型统一实现，无功能断层。

## 五、技术架构设计

### 1. 整体架构分层（从下至上，模块化低耦合，单模型核心架构）

- **硬件适配层**：负责手机 NPU/GPU/CPU 硬件加速适配，封装各芯片厂商硬件加速接口（Qualcomm SNPE/NNAPI/Core ML），为单模型 Gemma 3n E2B 推理提供统一硬件支撑；适配不同机型硬件差异，做量化版本定制化适配（低配机 INT4、标准机 INT8），确保单模型在各类手机上高效稳定运行；剔除多模型硬件加速适配接口。
- **模型层**：仅包含单模型 Gemma 3n E2B（INT4/INT8 分级量化版本），统一承载核心对话、翻译、记忆处理、精准指令识别、文字增强、2K 修图、ASR 语音识别、TTS 语音合成所有核心能力；采用 INT4/INT8 分级量化压缩技术降低模型体积和内存占用，提升手机端推理速度；实现单模型量化版本按需加载 + 后台轻量驻留机制，无模型切换、无多模型同时运行，避免发热、卡顿、内存溢出；剔除所有独立轻量工具模型层，原工具能力全部整合至单模型层。
- **核心服务层**：封装各类核心能力，提供统一标准化接口供上层调用，模块间低耦合，便于开发 / 测试 / 迭代；剔除多模型调度、工具模型调用相关服务模块，简化服务层结构：
  - **对话服务**：负责双模式对话逻辑、AI 主动交互触发、上下文串联、呼吸感逐字输出速度自动调节；基于单模型流式推理实现，无多模型数据拼接
  - **记忆服务**：负责轻量向量库 + SQLite 双引擎存储、记忆检索 / 召回（不做清洗 / 分级）；记忆处理与单模型推理无缝衔接
  - **工具服务**：负责单模型内工具能力的调用、功能实现（2K 修图、文字增强、语音交互），衔接对话服务与模型层，处理精准指令触发逻辑；无独立工具模型调用逻辑，仅为单模型能力分发
  - **合规服务**：负责权限申请 / 管理（含统一话术弹窗）、数据 AES-256 加密 / 解密、日志记录 / 删除 / 导出、隐私保护相关逻辑实现
  - **开关服务**：新增独立轻量模块，负责喇叭 / 小铃铛开关的状态切换、记忆、声音 / 弹窗 / 震动控制
  - **量化版本管理服务**：新增轻量模块，负责检测手机硬件配置、自动切换 INT4/INT8 量化版本、量化版本本地更新，为单模型层提供版本支撑
- **UI 交互层**：基于极简设计理念，实现全屏沉浸式对话界面，封装顶部栏（含双开关）、对话区、输入栏等核心 UI 组件；处理用户所有交互操作（点击 / 输入 / 语音触发 / 模式切换 / 双开关操作），衔接核心服务层，确保交互流畅、响应迅速，视觉与功能同步；剔除多模型切换相关 UI 交互逻辑，仅做单模型内模式切换。
- **应用层**：整合各层能力，实现 APP 整体功能，包含启动逻辑、版本更新、异常处理、崩溃恢复等，确保 APP 全流程稳定运行，处理极端场景下的优雅降级；新增量化版本异常兜底逻辑（INT8 卡顿自动切 INT4）；为后续迭代功能（文件整理、4K 修图、应用锁）预留接口；剔除多模型管理、调度相关应用层逻辑。

### 2. 核心技术选型

#### （1）移动端开发技术
- **Android 端**：Kotlin 语言，基于 Jetpack Compose 开发，适配 Android 10.0 及以上版本，保证界面流畅、性能稳定，符合安卓原生交互习惯；
- **iOS 端**：Swift 语言，基于 SwiftUI 开发，适配 iOS 15.0 及以上版本，贴合 iOS 原生交互风格，兼容苹果生态；
- **跨平台适配**：采用原生开发 + 统一接口封装的方式，确保两端功能一致、交互体验统一，避免跨平台框架带来的性能损耗、兼容性问题；两端均仅做单模型 Gemma 3n E2B 适配，无多模型跨平台差异。

#### （2）AI 模型相关技术（核心单模型技术，新增量化压缩关键选型）
- **模型量化压缩**：采用 INT4/INT8 分级量化压缩技术对单模型 Gemma 3n E2B 进行量化，INT8 为标准落地版本（平衡精度与性能），INT4 为低配机兜底版本（牺牲轻微精度换更低内存 / 功耗）；量化后模型体积较原模型压缩 ≥50%，内存占用 ≤400MB（INT8）/≤250MB（INT4）；剔除多模型量化相关技术，仅聚焦单模型分级量化。
- **硬件加速**：集成 TensorFlow Lite（通用）、Core ML（iOS 专属）、NNAPI/Qualcomm SNPE（Android 专属），实现单模型 Gemma 3n E2B 推理的硬件加速，最大化调用手机 NPU/GPU，减少 CPU 占用，降低发热；为 INT4/INT8 版本做针对性硬件加速优化。
- **模型管理**：实现单模型 Gemma 3n E2B（INT4/INT8） 本地加密存储、MD5 完整性校验、量化版本按需加载、后台轻量驻留、本地离线更新逻辑，确保模型运行安全、高效、低功耗；剔除多模型管理、加载、释放相关逻辑。

#### （3）数据存储技术
- **向量数据库**：选用轻量级本地向量库（FAISS Lite/Chroma Lite），用于存储用户记忆数据，支持高效相似性检索，实现记忆串联，适配手机端低内存要求；砍掉优先级排序 / 清洗相关逻辑，简化检索流程；与单模型推理引擎直连，提升记忆召回效率。
- **关系型数据库**：采用 SQLite，用于存储聊天记录、用户设置、操作日志、双开关状态、量化版本选择记录等结构化数据，轻量无依赖，确保数据稳定存储、高效查询，支持加密存储；
- **数据加密**：全程采用 AES-256 加密算法，对所有本地存储的用户数据、单模型文件进行加密，实现数据加密存储 / 解密读取，保障数据隐私安全；实现数据一键清空、日志手动删除 / 本地导出功能，提升用户数据控制权。

#### （4）其他核心技术（单模型统一承载，剔除独立工具技术选型）
- **语音交互**：由单模型 Gemma 3n E2B 直接承载本地 ASR/TTS 能力，无需集成独立轻量 ASR/TTS 模型（如 PicoTTS/Whisper Tiny），实现本地语音识别 / 合成，无云端交互，支撑情感模式沉浸式交互，支持音色 / 语速调节；识别准确率按 ≥90% 优化，降低开发调试成本。
- **图片处理**：由单模型 Gemma 3n E2B 直接承载图片处理能力，结合 Android/iOS 原生图片 API，实现 2K 修图（划痕修复）、文档文字增强（去阴影 / 折痕 / 文字锐化），仅访问 APP 沙盒内图片，无跨应用图片访问；砍掉 4K 超清 / 模糊修复相关逻辑；剔除 PP-OCRv5-Slim 相关图片处理技术选型。
- **合规适配**：基于 Android/iOS 原生权限管理 API，实现权限申请 / 管理 / 禁用，配合本地加密 / 日志管理，适配全球主流隐私法规，满足 Google Play/App Store 上架要求；实现权限申请统一话术弹窗，保障息屏 / 后台 / 通话时的权限支持。

### 2. 核心开发任务拆解（40 天，按模块、按天分配）

#### （1）单模型分级量化与适配模块（第 1-8 天，共 8 天）

- 第 3-4 天：集成 TensorFlow Lite/Core ML/NNAPI 硬件加速接口，为 INT4/INT8 版本做定制化硬件加速适配，完成接口调试，确保接口可正常调用，实现单模型不同量化版本的硬件加速推理；
- 第 5-6 天：实现单模型 Gemma 3n E2B（INT4/INT8） 本地加密存储、MD5 完整性校验逻辑，开发单模型量化版本按需加载 + 后台轻量驻留接口，确保单模型全程驻留、无模型切换；
- 第 7-8 天：适配 3 款海外主流中低端机型（Android 2 款、iOS 1 款），测试不同机型量化版本自动切换效果，优化单模型推理逻辑，初步测试 INT8 版本推理响应 ≤500ms、INT4 版本 ≤800ms，无明显发热；验证单模型承载的文字增强 / 2K 修图 / 语音交互核心能力。

#### （2）对话与记忆模块（第 9-20 天，共 12 天）
- 第 9-10 天：搭建双模式对话逻辑框架，编写工作 / 情感模式 prompt，实现关键词触发模式切换，对接单模型 Gemma 3n E2B 推理接口；剔除多模型对话接口对接逻辑；
- 第 11-13 天：开发 AI 主动交互触发机制，编写提醒 / 话题延续算法（仅时间 + 事件关键词），实现交互频率可配置，调试主动交互功能，确保任务完成后自动弹窗、闹钟 / 事务提醒必响必弹；
- 第 14-16 天：搭建 SQLite 数据库，实现聊天记录 / 用户设置 / 双开关状态 / 量化版本选择记录存储，完成 AES-256 加密适配，开发数据一键清空功能；
- 第 17-19 天：集成 FAISS Lite/Chroma Lite 轻量向量库，实现记忆检索 / 召回核心逻辑（砍掉清洗 / 分级），完成跨会话记忆串联；将向量库与单模型推理引擎直连，优化记忆与对话的衔接；
- 第 20 天：开发 AI 消息呼吸感逐字输出功能（按硬件 + 量化版本自动调节速度），联调对话与记忆模块，测试跨会话上下文串联效果，修复记忆召回卡顿、不连贯问题。

#### （3）功能模块（翻译、工具、双开关）（第 21-30 天，共 10 天）

- 第 23 天：实现翻译 / 修图 / 文字增强等指令的精准识别逻辑，调试指令触发功能，确保指令可正常调起单模型对应能力，无模型切换；
- 第 24-25 天：基于单模型 Gemma 3n E2B 开发多语言离线互译功能，适配 8 种主流语言，优化翻译精准度，响应 ≤3s；INT4 版本做翻译精度兜底优化；
- 第 26-28 天：基于单模型 Gemma 3n E2B 开发本地 ASR 语音识别 + TTS 语音合成功能，实现音色 / 语速调节，语音识别按 ≥90% 准确率优化；剔除独立 ASR/TTS 模型集成，实现单模型语音交互能力闭环；

#### （4）UI 交互模块（第 31-37 天，共 7 天）

## 七、测试计划

### 1. 测试目标

确保 APP 所有核心功能正常实现、交互流畅，性能、稳定性、兼容性、安全性、合规性完全满足放宽后 / 新增需求，无重大漏洞 / 卡顿 / 崩溃，用户体验符合极简设计理念，符合 Google Play、App Store 上架要求；砍掉非刚需功能 + 多模型相关测试，减少测试工作量；重点测试单模型分级量化、量化版本切换、单模型工具能力落地效果，确保程序员开发无歧义、可直接落地。

### 2. 测试范围

- **功能测试**：覆盖所有核心功能（双模式对话、主动交互、记忆系统、多语言翻译、2K 修图、文字增强、双开关功能、量化版本自动 / 手动切换等），砍掉文件整理 / 4K 修图 / 模糊识别 / 多模型适配测试，验证功能实现与需求一致性，精准指令触发准确率 ≥95%；重点验证单模型是否可承载所有工具能力，无功能缺失；
- **性能测试**：全维度测试启动速度、单模型不同量化版本推理速度、内存占用、功耗、机身温度、响应时间，确保所有放宽后性能指标达标（INT8 内存 ≤400MB、INT4 ≤250MB、功耗 ≤20%、温度 ≤42℃等）；
- **兼容性测试**：测试海外主流中低端机型（Android/iOS 各 10 款，骁龙 6/7 系列、天玑 700/800 系列为主）、不同系统版本的适配情况，无界面错乱、功能异常、卡顿、闪退；重点验证不同机型量化版本自动切换效果，确保低配机修图 / 对话流畅；

- **合规性测试**：验证隐私合规、权限管理、数据存储、日志管理是否符合全球主流隐私法规及 Google Play/App Store 上架要求，重点验证权限话术弹窗、息屏 / 后台 / 通话权限、日志本地导出功能，无合规问题；
